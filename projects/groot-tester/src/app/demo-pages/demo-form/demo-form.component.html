Forms

We have created some simple groot components to simplify the most common usages. In case you need to do something more advanced, such as various validations, you can take a look at the Angular form validation guide and at the Bootstrap documentation for forms.

Customizing combo templates

If you need to render special items in the combo, we provide some forms of template customization. If you need to do something really fancy though, please use the raw ng-select component, rather than our wrapper. You might also want to check out the source code of ng-select to better understand how to use the templates.

You have the option to pass custom angular templates to the components:

  • itemTemplate is the template used to render a specific item in the dropdown
  • labelTemplate is the template used to render one selected item. It can be used both for single and multiple selections. In case of single selections, you might want to use the same template as itemTemplate to achieve the same look and feel; in case of multiple selections, though, you might want to use a different template to add the "x" to unselect an item. Please checkout the default template to see how it is done.
  • the combo will show multiple labelTemplate side by side in case of multiple selections. You can also pass a multiLabelTemplate to replace them all with just one template, that you can customize as necessary.

For the combo checkbox component, you can customize far less: you can only pass a itemTemplate that will be displayed alongside the existing checkbox.

Filter text manually or very large combo

Sometimes you might want to filter your combo text by some special field (generally because you have a special rendering template). Another possible scenario is that you need to render a combo box with a very large number of values, and thus you want to load items a few at a time.

To support the first scenario, the groot-combo supports a property [filterServerSide]="true". To support the second, it supports [fetchDataIncrementally]="true". Note that the second implies the first. In both these cases, you need to subscribe to the event (requestData), which will be emitted when the user enters some text for searching the combo or when she scrolls down and the combo needs to load more values. At this point, you need to load the data and set either [items] (when fetchDataIncrementally is false) or [itemsPage] (when it is true).

Filter server side, no incremental load
<groot-combo
  label="Filtered currencies"
  [filterServerSide]="true"
  (requestData)="loadCurrenciesFiltered($event)"
  [items]="currenciesFiltered"
></groot-combo>
Incremental loading and server side filter
<groot-combo
  label="Incremental loading"
  [fetchDataIncrementally]="true"
  [maxItemsAtATime]="10"
  (requestData)="loadCurrenciesPage($event)"
  [itemsPage]="currenciesPage"
></groot-combo>

Note: when you receive a PopoverDataRequest do not write code like this: 


  searchPopoverNeedsData(request: PopoverDataRequest) {
    this.myService.getDomainValues(...)
      .subscribe(request.domainSubject);

This would forward the values that your service loads which is good, but also the complete, which would break incremental loading. Rather do something like this: 


  searchPopoverNeedsData(request: PopoverDataRequest) {
    this.myService.getDomainValues(...)
      .subscribe(
        data => request.domainSubject.next(data),
        err => request.domainSubject.error(err));

Show only selected elements

When the list of items is large so that you have to scroll the dropdown, it can be useful to filter the list to see only those elements already selected. To do so groot-combo supports a property [toggleShowOnlySelected]="true" that adds a toggle in the dropdown header (moreover the headers already defined).

Show only selected toggle with local storage
<groot-combo
        label="Currencies listbox"
        name="checkBoxMultiple"
        [items]="manyCurrencies"
        [required]="true"
        [(ngModel)]="form.selectedCurrencies"
        (ngModelChange)="onModelChange('listbox checkbox currencies', $event)"
        [showAsListBox]="true"
        [checkboxes]="true"
        [multiple]="true"
        [toggleShowOnlySelected]="true"
></groot-combo>

This also works with [filterServerSide]="true". In such a case two additional properties are sent to server: a boolean showOnlySelected that indicates the value of the toggle and an array selected with the list of all selected elements

Show only selected toggle with server side load
<groot-combo
        label="Filtered currencies"
        [filterServerSide]="true"
        (requestData)="loadCurrenciesFiltered($event)"
        [items]="currenciesFiltered"
        [multiple]="true"
        [toggleShowOnlySelected]="true"
></groot-combo>

server request example:


      filterText: 'ae',
      pageNum: 0,
      pageLen: 999999,
      showOnlySelected: true,
      selected: ['ALL', 'AMD', 'AOA']AA
